# Skill: Souls & web3 agents

Signed, portable agent identities for NFTs — and how they plug into
ERC-8004 / ERC-8183 / ERC-8257.

## Fetch a soul

```bash
curl -H "X-API-Key: $KEY" \
  "https://api.moca.qwellco.de/v1/souls/1/0xCONTRACT/42"
```

→ `data.soul` = `{ content (SOUL.md markdown), version, contentHash
(keccak256), signature (EIP-191), signerAddress, ipfsCid, ipfsUrl }` plus
`nft`, `collection`, and a `verification` block. Collection listing:
`GET /v1/souls/:chainId/:contract?page&limit`.

## Verify (always, two checks, viem)

```ts
import { recoverMessageAddress, keccak256, toBytes } from "viem";
const ok =
  keccak256(toBytes(data.soul.content)) === data.soul.contentHash &&
  (await recoverMessageAddress({
    message: data.verification.message,   // "{hash}|{chainId}:{contract}:{tokenId}|codex-v{n}"
    signature: data.soul.signature,
  })).toLowerCase() === data.soul.signerAddress.toLowerCase();
```

No trust in the API needed — same verification works on the IPFS copy.

## Run it

The SOUL.md *is* the system prompt:

```ts
await llm.chat({ system: data.soul.content, messages: [...] });
```

Pair with `/v1/library/ask` for grounded knowledge and
`/v1/decc0s/:id?include=profiles` for structured persona data.

## The onchain agent stack (draft ERCs — verify current specs)

- **ERC-8004 Trustless Agents**: identity/reputation/validation registries.
  Register agents whose metadata points at the soul's `ipfsCid`; the
  signature chain proves the personality belongs to that token. Soulweaver
  reserves `mintedAgentId` + per-collection `agentRegistryAddress`.
- **ERC-8183 Agentic Commerce**: escrowed jobs (Open→Funded→Submitted→
  Terminal) with evaluator attestation. Reference the provider agent by its
  8004 id or soul coordinates in job descriptors.
- **ERC-8257 Agent Tool Registry**: onchain tool manifests with predicate
  access control. The MOCA API itself is a natural tool entry for
  8257-aware agents.

## Ownership

Souls are public; control is onchain. Gate holder-only actions via
`ownerOf(tokenId)` + a standard SIWE wallet signature. The MOCA API is
read-only by design — writes belong to contracts and wallets.

Souls are generated by Soulweaver (https://soulweaver.museumofcryptoart.com,
open source) for any ERC-721 collection: a per-collection world context
(canon, roles/factions, trait-enrichment tiers) built from community lore
(Cortex) grounds each token's on-chain traits into a codex → holder-woven
SOUL.md, signed and pinned.
